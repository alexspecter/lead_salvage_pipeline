PIPELINE USER GUIDE
===================

This guide explains how to use the polished Lead Cleaner Pipeline.

1. PREPARATION
--------------
Ensure you are in the project root and your virtual environment is active:
  source venv/bin/activate

2. STEP 1: DATA HEALTH ASSESSMENT (RECOMMENDED)
-----------------------------------------------
Before processing, run a "Dry Run" to assess data quality and predict AI usage.
No files will be written in this mode.

  python run_pipeline.py "your_data.csv" --health-check

- Success: Provides a report on "Ready" rows vs "Needs AI" rows.
- Security: Will still trigger the "Nuclear Protocol" if true malware is detected.

3. STEP 2: CLEANING YOUR DATA
-----------------------------
To run the full cleaning process (Phase 1, AI Phase 2, and Merge Phase 3):

  python run_pipeline.py "your_data.csv" "output_folder"

- If you don't specify an output folder, it defaults to "output/".
- The original row order of your file is strictly preserved.

4. STEP 3: REVIEWING OUTPUTS
----------------------------
The pipeline generates exactly 3 files per run in your output directory:

1. final_output_[run_id].csv
   - Your cleaned, verified leads.
   - Professional casing (e.g., "Director of HR", "VP of Engineering").
   - Acronyms (CEO, IT, MBA, etc.) are correctly uppercased.

2. rejection_cache_[run_id].csv
   - A single consolidated log of all "deleted" items.
   - Records rejected rows (duplicates/invalid), dropped columns, and sanitized values.

3. pipeline_log_[run_id].csv
   - Full technical audit trail for troubleshooting.

5. SECURITY FEATURES
--------------------
- SANITIZATION: If a cell starts with dangerous characters (=, +, -, @), the tool
  neutralizes it by prepending a single quote ('). DATA IS PRESERVED.
- DELETION: If true malicious patterns (e.g., "=cmd|") are detected, the file is
  deleted immediately and the program shuts down to protect your system.

6. ADVANCED: ANALYSIS
---------------------
To see a quick summary of your most recent run without opening CSVs:

  python analyze_results.py --latest
